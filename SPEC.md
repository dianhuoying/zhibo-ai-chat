# 智播助聊 — 产品技术规格 (SPEC)

> 版本: 1.0 | 日期: 2026-06-25 | 目标平台: Windows (C# WPF + WebView2)

---

## 一、产品定位

智播助聊是一款直播 AI 辅助工具。真人主播坐在镜头前正常直播，AI 作为副脑和副嘴，自动抓取直播间评论、生成符合主播人设的回复，并以克隆的主播音色朗读回复，注入直播间音频通道。

与数字人产品的核心区别：我们是增强真人，而非替换真人。

---

## 二、窗口基础属性

- 框架: C# WPF (.NET 9) + WebView2 渲染 UI
- 自定义 WindowChrome 无边框，圆角 12px
- 默认尺寸: 380x680
- 默认位置: 屏幕右侧吸附
- 深色主题底色: #0A0E17
- 支持拖拽移动、四角拉伸缩放、系统托盘最小化
- 标题栏折叠按钮可将窗口缩为 60x680 窄条模式

## 三、主界面层级结构 (6层)

### 第1层：标题栏

| 左侧：折叠箭头 + 产品名「智播助聊」
| 右侧：最小化 / 最大化 / 关闭 按钮
| 整栏支持鼠标拖拽移动窗口
| 折叠后窗口缩为窄条，仅保留状态灯 + 快捷回复

### 第2层：直播间连接栏

| 左侧状态灯：红色(未连接) / 青绿色常亮(已连接)
| 标签：「直播间地址：」
| 输入框：单行，placeholder 提示文字，圆角 8px，深灰底
| 连接按钮：默认「连接」，连接后文案变「断开」(暗红色)
| 已连接时输入框锁定禁用，点「断开」恢复初始状态
| 按钮旁显示当前平台名称（从下拉框联动）

### 第3层：实时评论流

- 深色底 + 毛玻璃效果，flex 列布局，column-reverse 使新评论置顶
- 每条评论左侧 3px 竖色条分类：
  - 绿色 = 提问类（AI 优先回复）
  - 蓝色 = 互动类（延后回复）
  - 灰色 = 无意义（自动忽略）
- 配色：用户名浅金色 #FFB347，高优白色 #E8ECF1，普通灰 #8892A4，低优深灰 #555F72
- hover 底色微亮，展示用户来访次数 tooltip
- 最多显示 80 条，超出自动移除旧评论
- 内嵌临时输入框：主播「接管麦克风」模式下可用，输入框 + 发送按钮，回车即发

### 第4层：AI 回复展示区

- 半透明卡片面板，比评论流底色浅一层
- AI 回复文本白色 13px，打字机逐字出现效果
- 底部状态行：
  - 语音模式：喇叭图标 + 旋转 loading + 「正在合成语音...」
  - 文字模式：「已推送」绿色勾号
- 技术说明：TTS 为全量返回，不显百分比进度，用不确定动画代替

### 第5层：底部控制栏

从左至右顺序排列：

- 人设下拉框：预置「游戏主播」「带货达人」「聊天陪伴」，右侧 + 号可自定义新增
- 音色按钮组（三个并排）：男声 / 女声 / 我的声音
  - 选中态高亮青蓝色边框 + 微弱外发光
  - 选中后自动播放该音色样板语音
- 静音快捷键提示：点击切换静音/取消，hover 显示 Ctrl+M
- 接管按钮（核心交互）：
  - 默认态：「接管麦克风」，灰底，AI 不发声，主播手动互动
  - 点击后：高亮青蓝色，文案变「AI 全自动」，AI 自动语音回复弹幕
  - 再次点击切回手动，纯二态切换，无中间档位
- 设置齿轮按钮：点击弹出设置面板

### 第6层：可折叠快捷回复面板

- 默认展开，点击折叠箭头收起为窄条
- 2-3 列自适应网格，圆角胶囊按钮，hover 微亮
- 预设 6-8 条高频回复，点击直接推送到直播间输入框
- 示例：「谢谢大哥的礼物」「欢迎新来的宝」「点个关注不迷路」「今天就这个价」「已经是最低价了」「马上讲解」「安排！」
- + 按钮：弹出弹窗添加自定义短句，持久化到本地 SQLite
- 右键长按可编辑或删除已有条目

## 四、设置面板（弹窗式，覆盖主窗口）

- 左侧竖排 Tab 栏：音色管理 | 数据统计 | 日志回放
- 右侧内容区根据选中的 Tab 切换
- 选中 Tab 高亮左侧青蓝竖条标识

### Tab 1：音色管理

- 两个预置音色卡片（男声/女声）+ 一个「我的声音」卡片
- 每个卡片：音色名称 + 试听按钮
- 「我的声音」卡片：
  - 未克隆：显示「未录制」+「开始录制」→ 进入录音向导
  - 已克隆：显示「已绑定」+ 试听 +「重新录制」

#### 录音向导（4步分步弹窗）

**步骤 1/4：环境检测**
- 自动检测环境噪音分贝值
- 显示是否合格，建议关闭风扇/空调
- 下一步按钮

**步骤 2/4：文本试读**
- 展示示范朗读文案
- 点击录音按钮录制 15 秒试读
- 可回放试听
- 下一步按钮

**步骤 3/4：正式录制**
- 展示涵盖不同语气的长文本
- 录制 5 分钟人声采样
- 进度条显示录制进度
- 完成录制按钮

**步骤 4/4：上传与训练**
- 上传音频到服务端
- 调用声音克隆 API（约 3-5 分钟）
- 进度条显示训练状态
- 完成后绑定专属 voice_id
- 每步支持返回上一步，失败自动重试一次
### Tab 2：数据统计

- 本场直播 AI 回复总条数
- 语音回复条数 / 文字回复条数
- 互动趋势图：评论量随时间变化折线图（Canvas 绘制）
- 弹幕分类占比：横向堆叠条形图（提问/互动/无效）
- 数据每 5 秒刷新，可点「重置」清空本场数据

### Tab 3：日志回放

- 按时间轴排列，每条评论 + AI 回复成对显示
- 每条 AI 回复可标记「不合适」→ 红色高亮 → 后端累积标记用于优化 Prompt
- 过滤掉的黑粉评论显示「已过滤，未触发自动回复」
- 「导出」按钮：导出为 JSON 文件到本地
- 「清空」按钮：清除当前所有日志

## 七、整体架构

三层架构：

1. C# WPF 客户端（WebView2 渲染 UI + NAudio 音频管理 + 弹幕抓取插件）
2. ASP.NET Core 服务端（WebSocket + REST API，部署云服务器）
3. AI 推理层（LLM API + TTS API，服务端中转，客户端不持有 API Key）

客户端仅做展示和音频播放，所有 AI 调用和用户认证经服务端统一管理。
